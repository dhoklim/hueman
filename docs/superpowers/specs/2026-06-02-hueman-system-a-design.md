# hueman System A — 인터랙티브 텍스트 스토리 엔진 설계 명세

- **작성일:** 2026-06-02
- **범위:** hueman 3대 시스템 중 **System A(스토리 엔진)** 초안. 영상이 아직 없으므로 **텍스트 장면으로 영상을 대체**한다.
- **출처:** `제안서.txt` (hueman AI Build Spec) + 손그림 분기표 + 브레인스토밍 합의

---

## 1. 개요 & 목표

hueman는 관람자가 한 인간의 일생(탄생→죽음)을 선택·체험하고, 끝에 자신의 표정으로 합성된 사진 모자이크와 위로 메시지를 마주하는 인터랙티브 감정 설치 예술이다. 전체는 세 시스템(A 스토리 / B 실시간 감정·색필터 / C 모자이크·메시지)으로 이루어진다.

**이 명세는 System A만 다룬다.** 목표:

- 영상 대신 **데이터(JSON)로 정의된 텍스트 장면 그래프**를 재생하는 웹 엔진을 만든다.
- 선택 → 누적 분기(상태값) → 다음 장면 변형/분기 → 서사 엔딩 → **공통 작품 엔딩**까지 처음부터 끝까지 한 번에 도는 체험을 구현한다.
- B(웹캠 감정·색필터)와 C(모자이크·집계 메시지)가 **나중에 코드를 갈아엎지 않고 끼워질 인터페이스 자리**를 미리 비워둔다.

### 비목표 (이번 초안에서 하지 않는 것)

- 실시간 웹캠 표정 인식·심박 센서 (System B)
- 사진 모자이크 합성 (System C) — 엔딩에 placeholder 자리만 둔다
- 물리 버튼/하드웨어 연동 — 키보드/클릭으로 대체
- 모든 분기 조합의 완전 작성 — "각 단계 대표 선택 1세트"로 시작

## 2. 분기 모델 — C·하이브리드

제안서의 "누적 분기"를 다음과 같이 표현한다:

- **기본은 직선 척추 + 상태 플래그.** 단계는 정해진 순서로 진행하고, 각 단계의 핵심 선택은 *상태값(flag)* 만 바꾼다. 이후 장면은 누적 상태에 따라 **변형(variant)** 을 골라 재생한다(톤·내레이션·배경색이 달라짐).
- **진짜 갈림길은 한 곳:** 청소년기 "담배 권유 → 일탈 / 모범". 일탈 누적이 임계치를 넘으면 **배드엔딩**으로 빠지고, 아니면 본선에 합류한다.
- **정답은 없다.** 어떤 선택도 틀리지 않으며, 서사가 어떻게 끝나든(배드엔딩 포함) **작품 엔딩은 모든 경로 공통**이다.

### 확정 분기표 (초안 콘텐츠)

| 단계 | 대표 장면 | 핵심 분기 | 결과 / 상태 변화 |
|---|---|---|---|
| 오프닝(공통) | 암흑→빛→아기 울음→가족의 손→노란빛 | 없음 | 시작 감정 = 기쁨·사랑(노랑) |
| 유아기 | 옹알이·배냇짓, 기어다니기, 첫 사고 | **운다 / 안 운다** | 운다=기쁨(박수받음) / 안 운다=무감각 |
| 유년기 | 유치원 등원, 받아쓰기, 체육대회 | **넘어졌을 때: 일으켜주면 / 혼자 일어남(회피)** | 일으켜주면 `warmth+1`(따뜻함) / 혼자=쓸쓸함, 페이드아웃 |
| 청소년기 | 짝꿍·수업·하교 | **① 고백받음: 사귄다 / 안 사귄다** | 사귄다 `dating=true`(설렘) / 안 사귄다(공부·덤덤) |
| 청소년기 | (위와 겹침) | **② 담배 권유: 일탈 / 모범** | 일탈 `deviation+`, 모범 본선. 일탈 누적 임계치↑ → **배드엔딩** |
| 청년기 | 연애, 술자리·MT, 휴학+알바 | **부모님과 의견 충돌: 갈등 / 적응** | 갈등 `conflict+1` / 적응 `adapt+1` (이후 톤을 크게 가름) |
| 중장년기 | 결혼 여부, 직장·창업, 모임 | **결혼 / 비혼** *(주: 제안서의 '부모님 장례식' 분기 대신 손그림의 결혼 분기를 채택)* | 톤 변형 |
| 노년기(공통) | 배우자 떠남 → 자식과의 거리 → 암 → 죽음 | (대체로 공통, 자식과의 거리만 소분기) | 누적 상태에 따라 내레이션 변형 |
| 엔딩 | 서사 엔딩(일반/배드) → **작품 엔딩(공통)** | — | 공통 내레이션 → 위로 메시지 → 🖼모자이크 자리 |

> 확인된 변경: 유년기 분기는 손그림의 **체육대회 달리기 넘어짐**을 채택(제안서의 자전거 대신). 중장년기는 **결혼/비혼** 채택. 노년기는 **공통 흐름**으로 닫는다.

## 3. 감정-색상 시스템

실시간 색 필터와 모자이크 양쪽에서 동일하게 쓰이는 표(제안서 4장).

| 감정 키 | 묶음 | 색 | HEX | 집계 카테고리 |
|---|---|---|---|---|
| `joy` | 기쁨·편안·뿌듯 | 노랑 | `#FFD23F` | 기쁨 |
| `sad` | 슬픔·그리움·외로움 | 파랑 | `#3B7DD8` | 슬픔 |
| `anger` | 화남·증오·억울 | 빨강 | `#E03131` | 분노 |
| `numb` | 무표정·무념·지루 | 검정 | `#1A1A1A` | 무감각 |
| `anxiety` | 불안·두려움·**설렘** | 주황 | `#FF8C2B` | 불안 |
| `surprise` | 놀람·신기 | 초록 | `#2FB873` | **집계 제외** |

- 복합: `joy+sad`(감동), `sad+anger`(억울함) — 필터 혼합.
- **실시간 필터 색(6+혼합)과 위로 메시지 집계 카테고리(기쁨·슬픔·불안·분노·무감각 5종+복합)는 일부러 다르다.** 놀람은 필터엔 쓰되 집계 제외.

System A 초안에서는 **각 장면이 `emotion`을 선언**하고 그 색으로 화면을 물들인다(= System B가 들어오기 전의 감정 소스). 나중에 B의 실시간 웹캠 감정이 이 색을 덮어쓴다.

## 4. 아키텍처 — 모듈 (각자 한 가지 일만)

```
[입력: 키보드/클릭]
      │
      ▼
   engine ── 현재 씬 id + 상태(flags) 보유, 선택 적용
      │
      ▼
  resolver ── 누적 상태로 다음 변형/분기 선택 + 배드엔딩 판정
      │
      ▼
     ui ── 전체화면 장면 렌더링 · 감정 색 틴트 · 선택 버튼
      │            ▲
      │            └── emotionColor (감정→색; B가 덮어쓸 자리)
      ▼
 experienceLog ── {씬, 감정, 시각, 선택} 타임라인 기록
      │            (C가 집계할 데이터 계약)
      ▼
[엔딩] 최빈 감정 집계 → comfortMessages → 위로 메시지 + 🖼모자이크 자리
```

| 모듈 | 책임 | 의존 |
|---|---|---|
| `main.js` | 부팅, 전체 흐름 제어(오프닝→단계→엔딩) | engine, ui |
| `engine.js` | 상태머신: `currentSceneId` + `flags`. 선택 적용 | story.json, resolver |
| `resolver.js` | 누적 상태 → 변형 장면 선택, 배드엔딩 임계치 판정 (순수 함수) | — |
| `ui.js` | 장면 렌더, 감정 틴트 적용, 입력(키보드/클릭) | emotionColor |
| `emotionColor.js` | 감정 키 → 색/필터 매핑. **B 연동 지점** | — |
| `experienceLog.js` | 이벤트 기록 + 최빈 감정 집계 → 위로 메시지. **C 연동 지점** | comfortMessages |
| `comfortMessages.js` | 위로 메시지 풀(제안서 6장) | — |

각 모듈은 잘 정의된 인터페이스로만 소통하고 독립적으로 테스트 가능하게 둔다. `resolver`와 `experienceLog`의 집계는 순수 로직이라 단위 테스트 대상.

## 5. 데이터 모델

### 5.1 장면 노드 (`content/story.json`)

```json
{
  "id": "teen_smoke_choice",
  "stage": "adolescence",
  "type": "choice",
  "text": "쉬는 시간, 친구가 슬그머니 담배를 내민다. \"한 대 할래?\"",
  "emotion": "anxiety",
  "choices": [
    { "label": "받아들인다", "effects": { "deviation": 1 }, "next": "teen_deviant_1" },
    { "label": "거절한다",   "effects": {},                "next": "teen_model_1" }
  ]
}
```

- `type`: `scene`(내레이션, `next`로 진행) · `choice`(분기, `choices`) · `ending`(`endingType`: `normal`/`bad`/`common`)
- `emotion`: 3장 감정 키 중 하나 → 화면 색 틴트
- `next`: 다음 씬 id. **변형 분기**가 필요하면 `next`를 객체로:
  ```json
  "next": { "variants": [
    { "when": { "warmth": ">=2" }, "to": "elder_warm" },
    { "default": true,             "to": "elder_plain" }
  ]}
  ```
- 씬 id 명명 규약: `stage_topic_variant` (예: `child_fall_helped`).

### 5.2 상태(flags)

| flag | 타입 | 의미 | 발생 |
|---|---|---|---|
| `deviation` | int | 일탈 누적 | 청소년기 일탈 루트 |
| `warmth` | int | 따뜻함 누적 | 유년기 일으켜주기 등 |
| `conflict` | int | 갈등(내가 원하는 길) | 청년기 |
| `adapt` | int | 적응(부모님 길) | 청년기 |
| `dating` | bool | 연애 여부 | 청소년기 고백 |

### 5.3 배드엔딩 규약

- 청소년기 일탈 루트의 `family_counsel` 노드에서 `deviation >= 2`(임계치, 조정 가능)면 `bad_ending` 씬으로.
- 배드엔딩은 노년기까지 가지 않고 중간에 끝나지만, **곧바로 작품 엔딩(공통 내레이션 → 위로 메시지 → 모자이크 자리)** 으로 이어진다. 서사 길이는 달라도 마무리 구조는 항상 같다.

### 5.4 체험 이벤트 (C 데이터 계약)

```json
{ "sceneId": "teen_smoke_choice", "emotion": "anxiety",
  "source": "scene", "durationMs": 4200, "timestamp": 1780000000000 }
```

- `source`: 지금은 `"scene"`. 나중에 B가 `"webcam"` 이벤트를 같은 형식으로 추가한다.
- 엔딩 집계: `surprise` 제외, 감정별 누적 `durationMs` 최대 카테고리 1개 선택. 1·2위 차가 작으면 **복합 감정** 메시지.

## 6. 체험 흐름 & UI

- **연출:** 전체화면, 어둡고 시네마틱. 중앙에 큰 내레이션 텍스트, 배경은 장면 `emotion` 색의 라디얼/그라데이션 틴트. 장면 전환은 느린 페이드.
- **진행:** 일반 장면은 클릭 / `Space` / `Enter`로 다음. 선택 장면은 큰 버튼 2개 등장(`1`/`2` 또는 `←`/`→`).
- **오프닝(공통):** 암흑 → 커지는 빛 한 점 → 아기 울음(텍스트/사운드 placeholder) → 노란빛.
- **엔딩:** 공통 내레이션("…그 모든 감정이, 너였다") → "결과물 생성 중…" → 요약 카드(최빈 감정 + 위로 메시지 + `🖼 [모자이크 자리 — System C]`).
- 몰입을 위해 단계 라벨은 은은하게/숨김. 사운드는 이번 초안에서 선택(placeholder).

## 7. 기술 스택 & 실행

- **Vite + 순수 JS**(프레임워크 없음). 텍스트 장면 엔진엔 충분히 가볍고, B(face-api.js/MediaPipe·웹캠은 localhost 필요)·C(canvas)를 그대로 얹기 좋음.
- **테스트:** Vitest로 순수 로직(resolver 변형 선택, 배드엔딩 임계치, 집계→메시지) 단위 테스트. UI는 실행으로 검증.
- **실행:** `npm install` → `npm run dev` → 브라우저 자동 오픈.

### 파일 구조

```
hueman/
  index.html
  package.json
  vite.config.js
  src/
    main.js          # 부팅 + 흐름 제어
    engine.js        # 상태머신
    resolver.js      # 변형/배드엔딩 로직 (순수)
    ui.js            # 렌더 + 입력
    emotionColor.js  # 감정→색  (B 연동 지점)
    experienceLog.js # 로그 + 집계 (C 연동 지점)
    comfortMessages.js
  content/
    story.json       # 장면 그래프 + 분기표
  styles/
    main.css
  test/
    resolver.test.js
    experienceLog.test.js
```

## 8. B·C 연동 인터페이스 (미래 대비, 지금은 비워둠)

- **감정 enum/색표** (3장): A·B·C 공용 상수로 한 곳(`emotionColor.js`)에 둔다.
- **감정 이벤트** `{ emotion, source, durationMs, timestamp }`: A는 `source:"scene"`, B는 `source:"webcam"`을 같은 큐에 넣는다.
- **체험 로그 → 집계**: C는 `experienceLog`의 집계 결과(+ B의 실제 스냅샷)를 받아 모자이크를 만든다. A 단계에선 모자이크 자리에 placeholder만.

---

## 9. 콘텐츠 범위 (초안)

- 분기표(2장)대로 오프닝 + 6단계 + 노년기(공통) + 엔딩.
- 각 단계 대표 텍스트 장면 1~3개 + 핵심 분기. 내레이션은 실제 한국어로 작성(초안 품질, 이후 다듬기).
- 일탈 루트는 배드엔딩까지 한 경로를 실제로 완성해, "서사 길이는 달라도 작품 엔딩은 같다"를 보여준다.

## 10. 열린 질문 / 향후

- 사운드(아기 울음 등) 포함 여부 — 초안은 placeholder, 이후 결정.
- 변형 분기를 몇 군데까지 실제 구현할지 — 최소 노년기 1곳 + 엔딩 톤에서 시연.
- 내레이션 텍스트 톤·분량 — 작성 후 함께 다듬기.
