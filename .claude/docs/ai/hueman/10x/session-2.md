# 10x Analysis: hueman
Session 2 | Date: 2026-06-11
(이전 세션: `.Codex/docs/ai/hueman/10x/session-1.md`, 2026-06-08)

## Current Value

hueman은 3대 시스템(A 분기 스토리 / B 실시간 감정→색 / C 포토 모자이크)이 모두 동작하는 전시용 프로토타입이다. 세션 1의 "Do Now" 항목은 사실상 전부 구현 완료됐다: 감정 타임라인, 결과 카드 저장, opt-in 갤러리, 모자이크 색 매칭 개선, 프라이버시 안내, 사운드 cue, Shift+R 리셋, 작은 화면 레이아웃. 2026-06-11에는 `영상분기모음.docx`의 전체 분기 트리(돌잡이 6지선다, 소개팅 3지선다, grit/deviated/reformed 플래그, 엔딩 2종)로 스토리를 전면 교체했다.

핵심 가치는 변하지 않았다: **선택 → 색 → 마지막에 내 얼굴(모자이크)로 돌아오는 감정 루프.** 관람자는 "그 모든 감정이, 너였다"는 메시지를 몸으로 통과한다.

## The Question

프로토타입은 완성됐다. 이제 무엇이 이것을 **전시장에서 살아남고, 관람자가 친구를 데려오게 만드는 작품**으로 바꾸는가?

원 제안서(제안서.txt)와 현재 구현의 격차가 답의 절반이다:

| 제안서 명세 | 현재 상태 |
|---|---|
| 씬마다 영상 재생 | 텍스트만 (영상 트리는 docx에 준비됨) |
| 부스 1인 체험, 물리 버튼, 자동 흐름 | 브라우저 앱, 키보드/마우스, 수동 리셋 |
| 결과물 출력/전송해서 가져가기 | 부스 PC에 PNG 다운로드 (관람자 손에 안 들어감) |
| 심박 센서 각성도 보조 신호 | 미구현 (제안서에서도 선택 사항) |
| 복합 감정 필터 혼합 (노랑+파랑 등) | 단일 감정 색만 |

## Massive Opportunities

### 1. 영상 씬 레이어 (Video Scene Layer)
**What**: 씬 메타데이터에 `media` 필드를 추가하고, 영상이 있으면 배경에서 재생 + 텍스트/틴트 오버레이, 없으면 지금처럼 텍스트-only로 동작하는 점진적 구조. `영상분기모음.docx` 트리는 이미 story.json에 반영되어 있으므로 씬 ID ↔ 영상 파일 매핑만 추가하면 된다.
**Why 10x**: 제안서의 정체성 자체가 "인터랙티브 스토리 **영상**"이다. 텍스트 씬은 플레이스홀더라고 제안서 7장에 명시되어 있다. 영상이 들어오는 순간 이 프로젝트는 데모에서 작품이 된다.
**Unlocks**: 팀이 영상을 한 편씩 찍을 때마다 즉시 교체 투입 가능. 전부 못 찍어도 작품이 성립한다(혼합 모드).
**Effort**: Medium (재생 구조) + High (영상 제작은 코드 밖 작업)
**Risk**: 영상 로딩/프리로드 관리. 다음 씬 후보를 미리 프리로드하는 설계 필요.
**Score**: 🔥 Must do — 작품의 원래 정의를 완성하는 일

### 2. 전시 키오스크 모드 (Attract Loop + 자동 리셋)
**What**: (a) 무입력 시 감정 색들이 천천히 흐르는 어트랙트 화면 → 아무 키나 누르면 인트로, (b) 결과 화면에서 N초 무입력 시 자동 리셋, (c) 시작 전 카메라 준비 체크(얼굴 검출/조명 OK 표시), (d) 전체화면 + 커서 숨김.
**Why 10x**: 전시는 운영자가 자리를 비우는 시간이 대부분이다. 지금은 관람자가 결과 화면에 머문 채 떠나면 다음 사람이 남의 결과를 본다(감정 데이터 노출이기도 하다). 어트랙트 화면은 멀리서 관람자를 끌어들이는 호객이기도 하다.
**Unlocks**: 무인 운영, 회전율, 프라이버시 보장.
**Effort**: Medium
**Risk**: 자동 리셋 타이밍이 너무 짧으면 결과를 음미하는 관람자를 쫓아낸다. 60–90초 + "손 흔들면 연장" 같은 완충 필요.
**Score**: 🔥 Must do — 공개 전시 전 필수

### 3. 리빙 갤러리 월 (집단 감정 벽) — 세션 1에서 이월
**What**: 두 번째 화면(별도 브라우저 창, `?wall` 모드)이 opt-in 갤러리에 쌓인 결과 카드들을 천천히 흐르는 콜라주로 투사. 하루 동안 벽의 지배 색이 그날 관람자들의 감정 분포로 변한다. localStorage + BroadcastChannel이면 서버 없이 같은 PC의 두 창으로 구현 가능.
**Why 10x**: 메시지가 "그 모든 감정이 나였다"에서 **"우리 모두의 감정이 여기 함께 있다"**로 확장된다. 체험을 마친 관람자가 자기 카드가 벽에 걸리는 순간을 보러 남는다 — 전시의 피날레가 생긴다.
**Unlocks**: 전시 규모감, 체험하지 않은 관람자도 볼거리가 생김, 심사/도슨트 설명 포인트.
**Effort**: Medium (인프라가 이미 절반 — 갤러리 저장 구현됨)
**Risk**: opt-in을 명확히 — "갤러리에 남기기"가 벽 투사 동의임을 문구로 표시.
**Score**: 🔥 — 세션 1 평가 그대로, 이제는 기반(갤러리)이 깔려서 effort가 내려갔다

## Medium Opportunities

### 1. 결과물을 관람자 손에 — QR 가져가기
**What**: 결과 화면에 QR 코드를 띄워 관람자 폰으로 결과 카드를 가져가게 한다. 로컬 네트워크 미니 서버(전시 PC에서 `vite preview` 옆에 간단한 업로드 엔드포인트)에 카드를 올리고 짧은 URL을 QR로 표시. 오프라인 대안: 부스 프린터로 즉석 출력(제안서 [5] 원안).
**Why 10x**: 지금 "저장"은 부스 PC 다운로드 폴더로 간다 — 관람자는 빈손으로 떠난다. 작품의 핵심 약속이 "가져가기"인데 이 마지막 1미터가 끊겨 있다. 폰에 들어간 카드는 SNS로 퍼진다 — 작품의 입소문 그 자체.
**Impact**: 체험의 완결 + 자발적 홍보.
**Effort**: Medium (로컬 서버 + QR 라이브러리)
**Score**: 🔥 Must do

### 2. 오늘의 감정 통계 — "당신은 오늘 N번째 인생"
**What**: localStorage에 일별 익명 통계(체험 수, 최다 감정 분포)를 쌓고, 결과 화면에 한 줄 추가: "오늘 이 부스를 지나간 23번의 인생 중, 당신처럼 기쁨이 가장 컸던 인생은 9번이었습니다."
**Why 10x**: 한 줄의 텍스트로 솔로 체험에 집단의 맥락이 생긴다. 리빙 갤러리 월의 미니 버전을 거의 공짜로 얻는다. 갤러리 opt-in과 달리 익명 카운트라 동의 부담도 없다.
**Impact**: 결과 화면의 정서적 무게 상승, 재방문 유도("오후에 다시 와서 분포가 변했는지 봐야지").
**Effort**: Low–Medium
**Score**: 🔥 — 효과 대비 비용이 가장 좋은 항목

### 3. 복합 감정 색 혼합 (제안서 4장 미구현분)
**What**: 1·2위 감정이 비등할 때(이미 `isComposite` 판정 로직 존재) 틴트를 두 색의 그라디언트 혼합으로. 실시간 필터에서도 smooth 히스토리상 2개 감정이 교차하면 혼합 색 표시.
**Why 10x**: "기쁨+슬픔=감동, 슬픔+화남=억울함"은 제안서의 가장 시적인 디테일이다. 감정이 단색이 아니라는 작품 메시지를 색 시스템 자체가 증명하게 된다.
**Impact**: 작품의 개념적 정합성 + 시각적 다양성.
**Effort**: Medium (emotionColor에 혼합 그라디언트 추가, aggregate는 이미 준비됨)
**Score**: 👍 Strong

### 4. 심박 센서 각성도 입력 (Web Bluetooth)
**What**: BLE 심박계(표준 Heart Rate Profile)를 Web Bluetooth로 연결, 각성도(심박 변화율)를 표정 감정의 보조 신호로: 각성 높음 → 불안/설렘 쪽 가중, 낮음 → 무감각/편안 쪽 가중.
**Why 10x**: 표정은 숨길 수 있지만 심박은 못 숨긴다 — "당신 몸이 느낀 것"이라는 서사가 생긴다. 제안서 원안이기도 하다.
**Impact**: 체험의 신체성, 기술적 차별화.
**Effort**: Medium–High (하드웨어 확보 + Chrome 한정 + 착용 UX)
**Risk**: 센서 착용이 입장 절차를 늘리고 고장 포인트가 된다. 전시 안정성과 상충.
**Score**: 🤔 — 하드웨어가 이미 있으면 👍, 없으면 보류

## Small Gems

### 1. 돌잡이 선택을 인생의 복선으로
**What**: 돌잡이 6지선다(현재 effects 없음 — story.json:14-19)를 flag로 기록하고, 결과 카드나 노년 씬에 한 줄 콜백: "붓을 잡았던 아이는, 끝내 자신의 색으로 인생을 칠했습니다."
**Why powerful**: 이미 만든 textVariants 메커니즘 재사용. 첫 선택이 마지막에 돌아오는 수미상관 — 관람자가 소름 돋는 지점은 보통 이런 데서 나온다.
**Effort**: Low
**Score**: 🔥

### 2. 갤러리 저장 용량 보호
**What**: 결과 카드를 갤러리에 저장할 때 축소(절반 해상도 JPEG) + `setItem` quota 초과 시 오래된 항목 삭제 후 재시도.
**Why powerful**: 전시 하루 종일 돌리면 localStorage 5MB는 실제로 터진다. 리빙 갤러리 월을 하려면 더더욱 선결 과제.
**Effort**: Low
**Score**: 🔥 — 전시 신뢰성 버그에 가까움

### 3. face-api 동적 로드 (코드 스플리팅)
**What**: "카메라 켜고 시작" 클릭 시에만 `import('./liveEmotion.js')`. 초기 번들 1.34MB → 수십 KB.
**Why powerful**: 어트랙트 화면/인트로가 즉시 뜬다. 웹 포트폴리오 배포 시에도 필수.
**Effort**: Low
**Score**: 👍

### 4. 인트로 키 흐름 정리
**What**: (a) 인트로에서 Space/Enter가 무조건 카메라 시작으로 가는 것(ui.js:259)을 명시적 버튼 포커스 방식으로, (b) 선택지 ←/→ 키를 2지선다일 때만 활성화(현재 6지선다에서 의미 어긋남).
**Why powerful**: 관람자가 무심코 누른 키 하나로 권한 팝업이 뜨면 부스 흐름이 깨진다.
**Effort**: Low
**Score**: 👍

### 5. 체험 영상 캡처용 데모 모드
**What**: D 패널에 "자동 플레이" 토글 — 랜덤 선택으로 전 과정을 자동 진행. 포트폴리오 영상 촬영, 전시 전 스모크 테스트, 어트랙트 화면 소스로 삼중 활용.
**Why powerful**: 제출물/홍보 영상을 몇 번이고 다시 찍을 수 있고, 전 분기 경로 자동 점검도 된다.
**Effort**: Low–Medium
**Score**: 👍

## Recommended Priority

### Do Now (전시 신뢰성 + 저비용 고효과)
1. **갤러리 용량 보호** — 전시 중 터지는 유일한 시한폭탄. 리빙 월의 선결 과제.
2. **오늘의 감정 통계 한 줄** — 가장 싼 값에 작품의 정서적 스케일을 키운다.
3. **돌잡이 콜백** — textVariants 재사용, 콘텐츠 한 줄로 수미상관 완성.
4. **인트로 키 정리 + face-api 동적 로드** — 부스 흐름과 첫 로딩.

### Do Next (작품을 '전시'로 만드는 것)
1. **키오스크 모드** (어트랙트 루프 + 자동 리셋 + 카메라 준비 체크) — 무인 운영의 조건.
2. **QR 가져가기** — "가져가기"라는 제안서의 약속을 완성. SNS 확산 경로.
3. **영상 씬 레이어** — media 필드 + 점진적 교체 구조 먼저 만들고, 영상은 팀이 준비되는 대로 투입.

### Explore (전략적 베팅)
1. **리빙 갤러리 월** — 갤러리 기반이 깔린 지금이 적기. 두 번째 화면만 확보되면 서버 없이 가능.
2. **복합 감정 색 혼합** — 개념적 완성도. 영상 레이어보다 먼저 해도 좋다.
3. **심박 센서** — 하드웨어 확보 여부 확인 후 결정.

### Backlog
1. 분기 추가 확장 — 트리는 이미 docx 전체가 반영됨. 지금 경로가 충분히 깊다.
2. 모바일 단독 배포 — 전시가 끝난 뒤 포트폴리오 단계에서.
3. 무거운 시각 효과 — 감정 메시지를 강화할 때만.

## Questions

### Answered
- **Q**: 세션 1의 우선순위는 어디까지 소화됐나? **A**: Do Now 전체 + Do Next 일부(사운드, 콜백 일부)가 완료. 미소화분은 키오스크 모드, 리빙 월, 심박.
- **Q**: 스토리 콘텐츠가 더 필요한가? **A**: 아니다. docx 트리 전체가 반영됐고, 남은 건 콘텐츠가 아니라 매체(영상)와 운영(부스).

### Blockers (사용자 결정 필요)
- **Q**: 전시 형태 — 단독 부스(차광막+좌석, 제안서 원안)인가, 오픈 테이블인가? 키오스크 모드의 설계가 달라진다.
- **Q**: 전시장에 두 번째 화면(프로젝터/모니터)을 쓸 수 있나? → 리빙 갤러리 월 실행 여부.
- **Q**: 영상 촬영 일정이 잡혀 있나? → 영상 씬 레이어를 Do Next에 둘지, 구조만 미리 만들지.
- **Q**: 부스에 프린터 또는 로컬 Wi-Fi를 둘 수 있나? → QR vs 인쇄 선택.
